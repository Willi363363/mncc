# MNC Usage Guide

Learn how to use the MNC compiler by example.

## Compilation Command

```bash
./mncc <source_file> [output_file]
```

- `<source_file>`: Path to C source file (required)
- `[output_file]`: Path to output NASM file (optional, defaults to `output.asm`)

## Building MNC

### Prerequisites

- GCC or Clang
- Make
- `nasm` (for assembling the generated code)
- `ld` (linker)

### Build Steps

```bash
cd mncc/
make
```

This produces the `./mncc` binary.

## Creating a Simple Program

### Step 1: Write C Source

Create `hello.c`:
```c
int main() {
    int x;
    x = 42;
    return x;
}
```

### Step 2: Compile with MNC

```bash
./mncc hello.c
```

Output: `output.asm`

### Step 3: Assemble

```bash
nasm -f elf64 output.asm -o output.o
```

Creates `output.o` (object file).

### Step 4: Link

```bash
ld output.o -o hello
```

Creates `hello` executable.

### Step 5: Run

```bash
./hello
echo $?  # prints exit code: 42
```

## Example: Add Function

Create `add.c`:
```c
int add(int a, int b) {
    int result;
    result = a + b;
    return result;
}

int main() {
    int sum;
    sum = add(10, 32);
    return sum;
}
```

Compile and run:
```bash
./mncc add.c
nasm -f elf64 output.asm -o output.o
ld output.o -o add
./add
echo $?  # should print 42
```

## Example: Conditional

Create `conditional.c`:
```c
int abs(int x) {
    if (x < 0) {
        return -x;
    } else {
        return x;
    }
}

int main() {
    return abs(-15);
}
```

## Example: Loop

Create `factorial.c`:
```c
int factorial(int n) {
    int result;
    int i;
    result = 1;
    i = 1;
    while (i <= n) {
        result = result * i;
        i = i + 1;
    }
    return result;
}

int main() {
    return factorial(5);  // returns 120
}
```

## Inspecting Generated Assembly

The output NASM file can be inspected:

```bash
./mncc add.c
cat output.asm
```

Example output:
```nasm
; Generated by MNC
section .text
global main

add:
    push rbp
    mov rbp, rsp
    
    mov rax, [rbp + 16]    ; a
    push rax
    mov rax, [rbp + 24]    ; b
    pop rbx
    add rax, rbx
    mov [rbp - 8], rax     ; result = a + b
    
    mov rax, [rbp - 8]     ; return result
    
    pop rbp
    ret

main:
    push rbp
    mov rbp, rsp
    
    mov rdi, 10            ; first argument
    mov rsi, 32            ; second argument
    call add
    
    pop rbp
    ret
```

## Custom Output Filename

```bash
./mncc source.c my_asm.asm
nasm -f elf64 my_asm.asm -o my_asm.o
```

## Common Workflow with Bash Script

Create `compile.sh`:
```bash
#!/bin/bash

SOURCE=$1
BASENAME="${SOURCE%.c}"

# Compile C → NASM
./mncc "$SOURCE"

# Assemble NASM → Object
nasm -f elf64 output.asm -o output.o

# Link Object → Binary
ld output.o -o "$BASENAME"

echo "Build complete: ./$BASENAME"
echo ""
echo "Running:"
./"$BASENAME"
echo "Exit code: $?"
```

Usage:
```bash
chmod +x compile.sh
./compile.sh add.c
```

## Error Handling

### Syntax Error

Input: `syntax_error.c`
```c
int main() {
    int x
    return x;  // missing semicolon above
}
```

Running:
```bash
./mncc syntax_error.c
# Error: unexpected token TOK_RETURN at line 3
```

### Undeclared Variable

Input: `undeclared.c`
```c
int main() {
    return x;  // x not declared
}
```

Running:
```bash
./mncc undeclared.c
# Error: variable 'x' used but not declared
```

## Advanced: Multiple Files Worth of Code in One

Since MNC only supports single files, write all functions in one file:

```c
int helper1(int x) {
    return x * 2;
}

int helper2(int x) {
    return x + 10;
}

int main() {
    int a;
    int b;
    a = helper1(5);
    b = helper2(a);
    return b;
}
```

## Testing Exit Codes

Since MNC doesn't support printing (no `printf`), use exit codes:

```c
int test_add() {
    if (add(2, 3) == 5) {
        return 1;  // success
    } else {
        return 0;  // failure
    }
}

int main() {
    return test_add();
}
```

Then:
```bash
./mncc test.c
nasm -f elf64 output.asm -o output.o
ld output.o -o test
./test && echo "PASS" || echo "FAIL"
```

## Debugging Generated Assembly

If the program doesn't behave as expected, examine the assembly:

```bash
./mncc buggy.c
cat output.asm | head -30  # see first 30 lines
objdump -d output.o        # disassemble object file
```

## Notes on Architecture

- All integers are 64-bit signed (`int`)
- Variables are either global or function-local
- No optimization passes
- No dead code elimination
- No register allocation beyond RAX, RBX

## Limitations

- **No I/O**: Can't read input or print directly (only exit code)
- **No standard library**: No `printf`, `strlen`, etc.
- **Single file only**: Can't link multiple object files
- **No recursion optimizations**: Stack may overflow on deep recursion
- **Fixed calling convention**: x86-64 System V ABI assumed

## Tips & Tricks

### Verify Generated Assembly Syntax

```bash
./mncc source.c
nasm -f elf64 output.asm -l output.lst  # -l for listing file
cat output.lst
```

### Create Simple Makefile for Testing

```makefile
SRC = test.c
ASM = output.asm
OBJ = output.o
BIN = $(SRC:.c=)

$(BIN): $(OBJ)
	ld $^ -o $@

$(OBJ): $(ASM)
	nasm -f elf64 $< -o $@

$(ASM): $(SRC)
	../../mncc $< $@

run: $(BIN)
	./$(BIN)
	@echo "Exit: $$?"

clean:
	rm -f $(ASM) $(OBJ) $(BIN)

.PHONY: run clean
```

### Compare with GCC Output

For learning, compile the same C with real GCC and compare:

```bash
gcc -S -O0 -masm=intel source.c
# generates source.s (AT&T syntax with -masm=intel for Intel syntax)
diff -u source.s output.asm  # compare
```

This shows what a real compiler generates vs. your MNC output.
